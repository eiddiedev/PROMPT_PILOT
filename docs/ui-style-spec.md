# UI 风格规范

## 总体气质

目标不是“常规 AI SaaS 控制台”，而是参考你给的图，做成一种：

- 黑白工业感
- 超强网格秩序
- 杂志封面式首屏
- 实验室 / 孵化器 / 档案系统感
- 只有极少量橙色作为行动信号

整体视觉关键词：

- editorial
- industrial
- brutal precision
- modular grid
- monochrome interface
- incubator dashboard

## 视觉核心

### 1. 大标题必须成为视觉主角

首页主标题不要是普通 SaaS 文案，而是像参考图里的 `PORT_FOLIO` 一样，做成超大、宽占比极高、带强烈识别感的视觉标题。

推荐文案方向：

- `PROMPT_ARCHIVE`
- `PROMPT_INDEX`
- `PROMPT_LAB`
- `PROMPT_SYSTEM`

设计要求：

- 全大写
- 超粗、方形、带切角或像素切分感
- 横向占据首屏核心区域
- 字距略收紧
- 不要使用圆润创业风字体

### 2. 整个页面必须建立“工程图纸式栅格”

不是单纯内容分栏，而是让背景、分区、卡片、表头、按钮都服从统一的细线网格。

具体表现：

- 页面底层有淡灰色纵横辅助线
- 每个模块边界用 1px 到 1.5px 细线切割
- 外层有明显的深色粗边框，像一块被装裱的面板
- 所有内容像被安放在一个精密信息板上

### 3. 配色必须极简

建议颜色变量：

```css
:root {
  --bg: #f3f0ea;
  --panel: #f7f4ef;
  --line: #d7d1c8;
  --line-strong: #2b2b2b;
  --text: #111111;
  --muted: #6f6a63;
  --accent: #ff6a13;
  --accent-soft: #ffd7bf;
}
```

规则：

- 90% 区域使用米白、浅灰、黑
- 橙色只用在 CTA、激活标签、小面积状态条
- 不要上大面积蓝紫渐变
- 不要玻璃拟态、发光阴影、紫色科技风

## 字体建议

使用两套字体即可：

1. 展示字体
- 用于首页巨大标题
- 方向：方形、机械、几何、带未来工业感
- 可尝试：`Orbitron`, `Michroma`, `Teko`, `Chakra Petch`
- 如果可以组合，主标题尽量做自定义 letter spacing 和 uppercase 处理

2. 功能字体
- 用于正文、标签、表头、数字
- 推荐：`IBM Plex Mono`, `Space Mono`, `Geist Mono`

规则：

- 标题和正文必须风格差异明显
- 数字、标签、小标题尽量偏等宽字体
- 普通段落不要太多，尽量碎片化信息呈现

## 布局结构

### 顶层结构

页面整体像一个被框住的大面板：

1. 外层背景是浅米色画布，带淡淡工程线
2. 中间有一个大圆角矩形主框，边框较粗
3. 主框内部再切成：
- 顶部导航条
- 巨大标题区
- 左上说明区
- 右上精选 Prompt 详情区
- 中部筛选与搜索条
- 中下三列 Prompt 卡片网格
- 底部 CTA 和次要信息区

### 首页推荐分区

1. Top Bar
- 左上品牌名 `PromptPilot / Trae Prompt Lab`
- 中间少量导航，不要太多
- 右上橙色按钮：`OPEN LIBRARY` / `TRY OPTIMIZER`

2. Hero Typographic Block
- 超大标题占主舞台
- 左侧有一小段说明文字
- 右侧是“本周精选 Prompt”或“优化结果预览”

3. Utility Strip
- 类别 tab
- 搜索框
- 排序控件
- 视图切换

4. Prompt Grid
- 3 列网格
- 卡片边框清晰
- 卡片内部像信息模块，不像 Dribbble 卡片

5. Bottom CTA
- 左侧一句带姿态感的短句
- 右侧两个行动按钮

## 组件设计语言

### Prompt 卡片

每张卡片要像“信息样本卡”：

- 顶部小标签：分类
- 中间标题
- 一句用途摘要
- 底部不是花哨按钮，而是数据行
- 数据行建议：`复制次数` `收藏数` `优化提升`
- 右下角可以放一个细小箭头，表示进入详情

### 详情面板

右侧大卡不是浮层弹窗，而是固定在版面里的“精选样本面板”。

结构建议：

- 小状态标签
- 标题
- 一句定位说明
- Prompt 预览片段
- 3 到 4 个关键指标
- 主要按钮：复制、保存、优化

### 筛选条

尽量做得像控制面板，不像电商筛选器：

- 小尺寸矩形 tab
- 细边框
- 激活时填充浅灰或橙色
- 搜索框保持扁平，不要高圆角胶囊感

## 交互与动效

动效必须有，但要克制。

建议保留这几种：

1. 首屏加载
- 标题与线框分层淡入
- 卡片按列轻微错位出现

2. 卡片 hover
- 轻微上移 2px
- 边框加深
- 右下角箭头出现位移

3. 切换分类
- 使用淡入淡出加轻微位置变化
- 不要做花哨 3D

4. 打开优化结果
- 右侧面板内容滑入
- 前后 Prompt 对比区按列展开

## 必须避免

- 默认紫色 AI 产品风
- 发光、毛玻璃、大面积渐变
- 太多阴影
- 标准 SaaS 数据大卡拼贴
- 过多圆角
- 太多 icon
- 太多长段落解释

## 页面文案风格

避免过度营销和浮夸承诺，整体文案应该像：

- 实验室系统
- Prompt 档案库
- 结构化工作台
- 开发者使用说明

推荐文案口吻：

- `Indexed prompts for coding and learning workflows`
- `Saved, compared, optimized`
- `Community-tested prompt structures`
- `Debug faster. Learn cleaner. Interview sharper.`

## 实现建议

- 桌面端优先，移动端退化为单列
- 不要依赖图片素材，主要靠排版、边框、栅格和数字信息撑起质感
- 如果做背景，请使用极淡网格和少量几何线稿
- 图标要克制，尽量用线性箭头、方框、加号、十字

## 对 TraeSolo 的一句话描述

请不要生成一个普通的 AI SaaS 仪表盘，而要生成一个“黑白工业栅格风的 Prompt 档案工作台”，像实验室 portfolio、技术目录册和信息样本板的混合体。
